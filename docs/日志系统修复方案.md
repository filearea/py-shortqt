# py-shortqt 日志系统修复方案

## 当前问题分析

### 1. 日志输出不完整

**问题**：
- `SystemLogger.debug()` 只有在 `debug_mode=True` 时才输出
- 大量 `print()` 语句不会写入日志文件
- TUI 的 `screen=True` 重定向 stdout，导致 `print()` 看不到

**影响**：
- 调试信息丢失
- 无法通过日志文件排查问题
- 用户看不到关键信息

### 2. 日志系统架构混乱

**当前架构**：
```
LogManager (单例)
├── SystemLogger (Python logging)
│   ├── File Handler (DEBUG)
│   └── Console Handler (WARNING)
├── MarketLogger (直接写文件)
└── TradingLogger (直接写文件)
```

**问题**：
- 多个日志系统并存，不统一
- `print()` 和 `logger` 混用
- 没有集中管理

---

## 修复方案

### 方案 A：统一使用 logging（推荐）

**优点**：
- 统一管理
- 支持日志级别
- 支持日志轮转
- 支持异步写入

**缺点**：
- 需要修改大量代码

**实施步骤**：

1. **修改 SystemLogger**
   - 移除 `debug_mode` 限制
   - 所有 `debug()` 调用都写入文件
   - 控制台输出改为 INFO 级别

2. **替换所有 `print()` 为 `logger.info()`**
   - 启动日志 → `logger.info()`
   - 调试日志 → `logger.debug()`
   - 错误日志 → `logger.error()`

3. **添加日志工具函数**
   ```python
   # src/utils/logger.py
   def get_logger() -> SystemLogger:
       return LogManager().system
   
   def info(msg: str):
       get_logger().info(msg)
   
   def debug(msg: str):
       get_logger().debug(msg)
   
   def error(msg: str):
       get_logger().error(msg)
   ```

4. **修改 TUI 配置**
   - `redirect_stdout=False`
   - `redirect_stderr=False`
   - 或者使用 `console=True`（非全屏模式）

---

### 方案 B：保留 print()，添加日志镜像（快速修复）

**优点**：
- 改动小
- 快速见效

**缺点**：
- 代码冗余
- 维护成本高

**实施步骤**：

1. **创建日志工具函数**
   ```python
   # src/utils/log.py
   from src.loggers import LogManager
   
   _log_manager = None
   
   def get_log_manager():
       global _log_manager
       if _log_manager is None:
           _log_manager = LogManager()
       return _log_manager
   
   def log(msg: str, level: str = 'info'):
       """同时 print 和写入日志"""
       print(msg)
       lm = get_log_manager()
       if level == 'info':
           lm.system.info(msg)
       elif level == 'debug':
           lm.system.debug(msg)
       elif level == 'error':
           lm.system.error(msg)
   ```

2. **替换关键 `print()` 为 `log()`**
   - 启动日志
   - 初始化日志
   - 错误日志

---

### 方案 C：混合方案（折中）

**实施步骤**：

1. **修改 SystemLogger**
   ```python
   # 移除 debug_mode 限制
   def debug(self, msg: str):
       self.logger.debug(msg)  # 总是写入文件
   ```

2. **保留 `print()`，但添加日志回调**
   ```python
   # 在 main_live.py 开头
   import sys
   
   class LogMirror:
       def __init__(self, logger):
           self.logger = logger
           self.terminal = sys.stdout
       
       def write(self, message):
           self.terminal.write(message)
           self.logger.info(message.strip())
       
       def flush(self):
           self.terminal.flush()
   
   sys.stdout = LogMirror(LogManager().system)
   ```

3. **修改 TUI 配置**
   - 不使用 `screen=True`
   - 或使用窗口模式

---

## 推荐方案

**采用方案 A（统一使用 logging）**，分阶段实施：

### 阶段 1：修复 SystemLogger（今天）
- 移除 `debug_mode` 限制
- 确保所有 `logger.info()` 和 `logger.debug()` 都写入文件

### 阶段 2：替换关键 print()（今天）
- 启动流程的 `print()` → `logger.info()`
- 初始化的 `print()` → `logger.info()`
- 错误的 `print()` → `logger.error()`

### 阶段 3：修改 TUI 配置（明天）
- 测试 `screen=False` 的效果
- 或使用窗口模式

### 阶段 4：清理剩余 print()（后续）
- 逐步替换所有 `print()` 为 `logger` 调用

## 日志级别规范

| 级别 | 用途 | 示例 |
|------|------|------|
| DEBUG | 调试信息 | API 请求参数、函数调用 |
| INFO | 一般信息 | 启动、初始化、状态变更 |
| WARNING | 警告信息 | 配置缺失、重试 |
| ERROR | 错误信息 | API 失败、异常 |
| CRITICAL | 严重错误 | 程序崩溃、数据丢失 |
