### 1. 24 小时交易统计

#### 1.1 功能描述

自动追踪完整交易轮次（开仓→平仓），实时统计：
- 开仓次数 / 平仓次数
- 胜率（盈利轮次占比）
- 盈亏比（总盈利 / 总亏损）
- 累计盈亏

#### 1.2 TUI 展示

```
┌─────────────────────────────────────────────┐
│  📊 24h 统计: 开仓5次 平仓4次 胜率75%      │
│  盈亏比2.3:1  累计盈亏+42.56U               │
└─────────────────────────────────────────────┘
```

#### 1.3 统计逻辑

| 项目 | 说明 |
|------|------|
| 统计周期 | 24 小时滚动窗口 |
| 开仓计数 | 每次开仓完成（`position` 从空→有） |
| 平仓计数 | 每次完全平仓（`position` 从有→空） |
| 胜率 | 盈利轮次 / 平仓轮次 |
| 盈亏比 | 总盈利 USDT / 总亏损 USDT（绝对值） |
| 累计盈亏 | 所有平仓轮次的净 PnL |

#### 1.4 核心代码

```python
# src/trading/live.py

def _update_trade_stats_24h(self):
    """更新 24 小时交易统计"""
    self.trade_stats_24h = {
        'open_count': self._open_count,        # 开仓次数
        'close_count': self._close_count,      # 平仓次数
        'win_count': self._win_count,          # 盈利次数
        'loss_count': self._loss_count,        # 亏损次数
        'win_rate': win_rate,                  # 胜率
        'total_win_usd': self._total_win_usd,  # 总盈利
        'total_loss_usd': self._total_loss_usd, # 总亏损
        'profit_loss_ratio': pl_ratio,         # 盈亏比
        'total_pnl': self._total_pnl,          # 累计盈亏
    }
```

### 2. 历史持仓面板

#### 2.1 功能描述

TUI 实时展示最近 10 条历史持仓，每条卡片 4 行信息：
- 第 1 行：方向（LONG/SHORT）+ 状态 + PnL
- 第 2 行：开仓价 × 数量 @ 时间 + 平仓价
- 第 3 行：手续费 + 资费 + 净利
- 第 4 行：空行（分隔）

#### 2.2 TUI 展示

```
┌─ 历史持仓 ────────────────────────────────────────────────────────┐
│ [LONG] 完全平仓  PnL: +14.56U  ───────────────────────────────── │
│ 开仓 2114.57 × 0.909  @ 05-21 14:36  平仓 2132.70               │
│ 手续费 0.42U  资费 0.08U  净利 14.06U                            │
│                                                                   │
│ [SHORT] 未平仓  PnL: --  ─────────────────────────────────────── │
│ 开仓 2124.98 × 0.020  @ 05-21 15:18                              │
│ 手续费 0.01U  资费 --  净利 --                                   │
│                                                                   │
│ ... (最多10条，超出的截取)                                         │
└───────────────────────────────────────────────────────────────────┘
```

#### 2.3 配色规则

| 元素 | 颜色 |
|------|------|
| LONG | 粗体绿色 |
| SHORT | 粗体红色 |
| 未平仓 | 黄色 |
| 完全平仓 | 白色（dim） |
| 部分平仓 | 青色 |
| PnL 正 | 绿色 |
| PnL 负 | 红色 |
| PnL 未平仓 | 黄色（--） |

#### 2.4 数据来源

通过币安 API `get_fills()` 分页拉取最近 N 天成交记录（每次 1000 条，上限 10000 条），
配合 `get_income_history()` 获取资金费率收入，按 LONG/SHORT 分组配对开仓/平仓生成持仓历史。

#### 2.5 核心代码

```python
# src/trading/live.py

async def fetch_position_history(self):
    """拉取历史持仓（分页拉取，上限 10000 条）"""
    # 1. 拉取成交记录
    fills = await self.api.get_fills(limit=1000, from_id=from_id)
    
    # 2. 拉取资金费率收入
    income = await self.api.get_income_history(symbol=self.symbol, limit=1000)
    
    # 3. 配对开仓/平仓
    positions = self._pair_positions(fills, income)
    
    # 4. 取最近 10 条
    self.position_history = positions[:10]
```

#### 2.6 更新频率

- 启动时拉取一次
- 主循环每 600 帧（~30 秒）刷新一次
- 每次持仓同步后刷新

### 3. 音效提醒

#### 3.1 功能描述

成交和关键事件自动播放提示音（Windows `winsound` + `ding.wav`），
无需紧盯屏幕即可感知交易状态变化。

#### 3.2 音效规则

| 事件 | 响声次数 | 说明 |
|------|----------|------|
| 开仓成交 | 1 声 | 挂单被吃，新持仓建立 |
| 开仓成交（部分） | 1 声 | 部分成交 |
| 平仓/止盈/止损 | 3 声 | 持仓结束 |
| 保底止损成交 | 3 声 | 同平仓 |
| 提前平仓成交 | 3 声 | 同平仓 |
| 移动止损成交 | 3 声 | 同平仓 |
| 市价全平 | 3 声 | 同平仓 |
| 持仓同步 | 3 声 | 交易所确认平仓 |
| 持仓超时保本 | 4 声 | 浮亏保护触发 |

#### 3.3 音效开关

系统设置 → 系统设置 tab → `sound.enabled`（开/关）

#### 3.4 技术实现

```python
# src/trading/live.py

def play_ding(self, count: int = 1):
    """播放提示音（count 控制响几声）"""
    if not self._ding_path:
        return
    if self.config_manager and not self.config_manager.is_sound_enabled():
        return
    try:
        import winsound
        for _ in range(count):
            winsound.PlaySound(self._ding_path, winsound.SND_FILENAME)
    except Exception:
        pass

def _trigger_sound(self, action: str):
    """根据 action 名称自动触发对应音效"""
    if '开仓成交' in action or '开仓成交（部分）' in action:
        self.play_ding(1)
    elif '持仓超时' in action:
        self.play_ding(4)
    elif any(kw in action for kw in [
        '止盈成交', '止损成交', '保底止损成交',
        '提前平仓成交', '移动止损成交',
        '市价全平', '开仓成交（部分）',
        '持仓同步'
    ]):
        self.play_ding(3)
```

订单簿买卖档数量一致，最新价永远居中显示，窗口大小变化时自动调整。

#### 4.2 自适应逻辑

```
top 区域可用高度 = 总高度 - header(3) - footer(12) - indicators(5) - Panel边框(4)
买卖盘档数 = min((top_height - 1) // 2, 20)   # 最多各 20 档
买卖盘档数 = max(档数, 5)                      # 最少 5 档
实际深度不足时用空行补齐
```

### 5. 买卖深度固定宽度

#### 5.1 功能描述

买卖盘深度值固定为 4 位整数 + 2 位小数（如 `0015.50 ETH`），不足部分 0 填充，
确保 TUI 不会因深度数据变化而频繁改变长度。

#### 5.2 势均力敌指示

当买卖深度差值 ≤ 15% 时，显示灰色"势均力敌"，替代具体的深度不平衡数值。

```
买盘占优（>15%）→ 绿色标记 "买盘占优"
卖盘占优（>15%）→ 红色标记 "卖盘占优"
差值 ≤ 15%     → 灰色标记 "势均力敌"
```

#### 5.3 格式示例

```
深度值 15.5   →  0015.50 ETH
深度值 125.3  →  0125.30 ETH
深度值 8.0    →  0008.00 ETH
深度值 1000.5 →  1000.50 ETH
```

# ATR 波动率百分比 = ATR(14) / 当前价格 × 100
atr_pct = (atr_14 / current_price) * 100

### 7. 部分成交处理

#### 7.1 功能描述

挂单部分成交时自动处理剩余数量，支持超时撤单容错。

#### 7.2 处理流程

```
挂单 → 部分成交 → 剩余数量 > 0
  → 继续等待剩余部分成交
  → 超时后撤单
  → 撤单成功 → sync_account()
  → 撤单失败（订单不存在）→ 静默跳过
```

#### 7.3 容错处理

币安条件单撤销时可能返回 `-2011`（订单不存在），此时静默跳过，
不抛出异常，避免影响后续流程。

## ⚠️ 注意事项

### 1. 音效系统

| 限制 | 说明 |
|------|------|
| Windows 专属 | 使用 `winsound` 模块，仅 Windows 系统有效 |
| 需要 ding.wav | `sounds/ding.wav` 文件必须存在 |
| 可关闭 | 系统设置 → 系统设置 → sound.enabled = false |

### 2. 历史持仓

| 限制 | 说明 |
|------|------|
| 分页拉取 | 每次 1000 条，上限 10000 条 |
| 配对逻辑 | 按方向分组，开仓→平仓配对，可能不完全精确 |
| 资费计算 | 基于资金费率收入记录，可能缺少部分数据 |

### 3. 24h 统计

| 限制 | 说明 |
|------|------|
| 滚动窗口 | 非严格 24 小时，程序运行期间累计 |
| 部分成交 | 部分成交的轮次可能在统计中存在偏差 |

### 4. 鼠标滚轮（已知问题）

| 问题 | 说明 |
|------|------|
| Windows Terminal 滚轮触发下单 | 现代 Windows Terminal 将鼠标滚轮转译为箭头键序列（`\xe0` + `H/P`），与真实方向键字节级一致，无法在代码层面区分 |
| 解决方案 | 建议在终端设置中禁用鼠标报告模式，或使用传统 conhost 终端运行 |
